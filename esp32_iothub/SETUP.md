# ESP32 Setup Guide

## Libraries (Arduino IDE Library Manager)

- `PubSubClient` by Nick O'Leary
- `ArduinoJson` by Benoit Blanchon

> `WiFi.h`, `WiFiClientSecure.h` and `time.h` are included with the ESP32 board package — no separate install needed.

---

## 1. Connection string

In Azure Portal → IoT Hub → Devices → your device, copy the primary connection string:

```
HostName=YOURHUB.azure-devices.net;DeviceId=YOURDEVICE;SharedAccessKey=abc123==
```

Map the three parts to the constants in the sketch:

```cpp
const char* IOT_HUB_HOST  = "YOURHUB.azure-devices.net";   // HostName
const char* DEVICE_ID     = "YOURDEVICE";                   // DeviceId
// SharedAccessKey is used to generate the SAS Token (see below)
```

---

## 2. Generate the SAS Token

The SAS Token is the MQTT password. It is generated by signing the SharedAccessKey with HMAC-SHA256.

**Option A — Azure CLI** (easiest, valid for 1 year):

```bash
az iot hub generate-sas-token \
  --hub-name YOURHUB \
  --device-id YOURDEVICE \
  --duration 86400
```

Copy the result (starts with `SharedAccessSignature sr=...`) and paste it into `SAS_TOKEN`.

**Option B — VS Code:**
Azure IoT Hub extension → right-click on the device → *Generate SAS Token*

> Note: The token has an expiry date. When it expires the device will fail to connect (MQTT state -4 or -5). Regenerate and reflash the firmware.

---

## 3. TLS root certificate

Azure IoT Hub uses certificates signed by **DigiCert Global Root G2**.

Download the PEM from:
```
https://cacerts.digicert.com/DigiCertGlobalRootG2.crt.pem
```

Paste the full content into the `ROOT_CA` variable in the sketch.

---

## 4. MQTT Username

Must follow this exact format:

```cpp
const char* MQTT_USERNAME = "YOURHUB.azure-devices.net/YOURDEVICE/?api-version=2021-04-12";
```

## Sending C2D commands

The device listens for JSON messages that change the telemetry send interval. This matches the Python example behaviour.

**From Azure CLI:**

```bash
az iot device c2d-message send \
  --hub-name YOURHUB \
  --device-id YOURDEVICE \
  --data '{"period": 5, "message": "period changed"}'
```

**From Azure Portal:**
IoT Hub → Devices → your device → **Message to Device** → write JSON in the body.

### C2D message format

| Field      | Type   | Description                                      |
|------------|--------|--------------------------------------------------|
| `period`   | int    | New telemetry interval in seconds                |
| `message`  | string | Text printed to Serial (confirmation / label)    |

**Example:**
```json
{"period": 3, "message": "faster interval"}
```

> The device updates `period` immediately on the next loop iteration. No restart needed.

---

## Telemetry payload

The device sends a JSON message on every interval with the following fields, mirroring the Python example:

```json
{
  "temperature": 27,
  "humidity": 65,
  "pressure": 1012,
  "when": "2025-01-15T10:30:00Z"
}
```

| Field         | Range       | Description                        |
|---------------|-------------|------------------------------------|
| `temperature` | 25 – 29     | Simulated temperature (°C)         |
| `humidity`    | 50 – 99     | Simulated humidity (%)             |
| `pressure`    | 900 – 1099  | Simulated pressure (hPa)           |
| `when`        | ISO 8601 UTC| Timestamp from NTP                 |

Serial output format (matches Python `print`):
```
27 65 1012
```

```bash
az iot hub monitor-events --hub-name YOURHUB --device-id YOURDEVICE
```

## Notes

- The telemetry topic includes `$.ct=application%2Fjson` so Azure stores the body as readable JSON instead of Base64.
- `mqttClient.loop()` in the main loop is critical — without it C2D messages are never received.
- Avoid long `delay()` calls in `loop()` as they delay incoming command processing.
- Port **8883** = MQTT over TLS. Port 1883 (unencrypted) is not accepted by Azure.
- The ESP32 syncs time via NTP on startup (`pool.ntp.org`). The `when` field will be incorrect if NTP sync fails — check Serial output for the sync step.
- The default send interval is **10 seconds**, matching the Python example. Change it via C2D without reflashing.
